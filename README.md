# 🌌 Web Starter — создавайте приложения, просто описав их словами

Это стартовый набор, с которым **любой человек без знания программирования** может создать
своё веб-приложение или сайт — прямо в приложении **Claude Code Desktop**. Вы не пишете код.
Вы просто описываете, что хотите, отвечаете на пару простых вопросов — и приложение появляется.

> Внутри уже лежит красивая стартовая страница и «умные команды» для Claude. Дальше — только
> ваша фантазия.

---

## Что вы умеете делать сразу

| Команда | Что делает |
| --- | --- |
| `/app-start` | Запускает приложение и показывает его в окне справа. Начните с неё. |
| `/app-init` | Создаёт ваше приложение: задаёт вопросы, предлагает дизайн, собирает его. |
| `/app-build` | Продолжает сборку, если она прервалась. |
| `/app-push` | Сохраняет проект на GitHub в один шаг (commit + push). |
| `/app-change-text` | Быстро меняет текст: выделите надпись в окне справа и напишите новую. |

## Шаг 1. Установите три программы (один раз)

1. **Node.js** — это «движок», на котором работают сайты.
   Откройте [nodejs.org](https://nodejs.org), скачайте версию с пометкой **LTS** и установите
   (просто нажимайте «Далее»).
   Видеоинструкция: https://youtu.be/7S7FjiZ-zq8?si=Yrx4gbKrWZnjuHkS
2. **Claude Code Desktop** — приложение, в котором вы будете общаться с Claude.
   Установите его и войдите в свой аккаунт.
3. **Git** - система версионирования проекта
    Видеоинструкция: https://youtu.be/LHuott_frOA?si=YADmRP-9wA13knpG

## Шаг 2. Скачайте этот проект

На странице проекта в GitHub нажмите зелёную кнопку **Code → Download ZIP**. Распакуйте архив
в удобную папку, например `Документы/мой-проект`.

## Шаг 3. Откройте папку в Claude Code Desktop

Запустите Claude Code Desktop и откройте папку, которую только что распаковали
(**Open Folder / Открыть папку**).

## Шаг 4. Посмотрите демо

В поле чата напишите:

```
/app-start
```

Через минуту справа откроется красивая страница. Это значит, что всё работает 🎉

## Шаг 5. Создайте своё приложение

Напишите команду и кратко опишите свою идею, например:

```
/app-init сайт для записи клиентов в мою кофейню
```

Дальше Claude задаст несколько простых вопросов (нужно будет просто выбирать варианты),
**предложит на выбор несколько вариантов дизайна** под вашу идею и, когда вы одобрите план,
**соберёт приложение**. Чтобы снова его посмотреть — напишите `/app-start`.

> 🎨 **Дизайн.** Среди вариантов всегда есть пункт «Как на конкретном сайте» — выберите его и
> вставьте ссылку на сайт, который вам нравится. Claude возьмёт оттуда стиль (цвета, шрифты,
> настроение) и применит к вашему приложению. Чужие тексты и картинки не копируются — только
> внешний вид.

## Шаг 6. Сохраните проект на GitHub (когда захотите)

Когда приложение нравится и хочется его сохранить в облаке — напишите:

```
/app-push
```

При первом вызове Claude попросит ссылку на ваш репозиторий на GitHub — заранее создайте пустой
репозиторий на github.com и вставьте его ссылку. Она выглядит примерно так:
`https://github.com/ваш-логин/название-репозитория.git`. После этого появится окно входа — выберите
в нём пункт **«Войти через браузер»** и войдите в свой аккаунт GitHub. Это нужно сделать один раз.
Все следующие `/app-push` будут просто сохранять изменения. Никаких git-команд вводить не нужно.
Ваши секретные ключи (`.env.local`) на GitHub **не попадают**.

При каждом сохранении Claude записывает, что именно поменялось, в журнал изменений
([`CHANGELOG.md`](CHANGELOG.md)) — там всегда видно историю проекта простыми словами.

## Шаг 7. Опубликуйте приложение в интернете через Vercel (когда захотите)

Пока ваш проект сохранён только на GitHub — увидеть его может лишь тот, кто открыл его на
своём компьютере. Чтобы у приложения появился настоящий адрес в интернете (например
`мой-проект.vercel.app`), подключите бесплатный сервис **Vercel** — он сам публикует сайт
из вашего GitHub-репозитория и обновляет его при каждом `/app-push`.

1. Откройте [vercel.com](https://vercel.com) и нажмите **Sign Up**.
2. На экране входа выберите **Continue with GitHub** и войдите тем же аккаунтом GitHub,
   куда вы сохраняли проект командой `/app-push`.
3. На главной странице нажмите **Add New... → Project**.
4. В списке репозиториев найдите свой проект и нажмите **Import**.
   Если репозиторий не виден — нажмите **Adjust GitHub App Permissions** и разрешите Vercel
   доступ к нему.
5. Vercel сам узнает, что это приложение на Next.js — ничего менять не нужно. Если у вашего
   приложения есть база данных или другие сервисы (например Supabase, Resend, Telegram,
   Stripe), откройте раздел **Environment Variables** и добавьте туда те же ключи, что лежат
   у вас в файле `.env.local` (так же, как они называются в этом файле).
6. Нажмите **Deploy** и подождите одну-две минуты.
7. Когда сборка закончится, Vercel покажет ссылку на ваш сайт — она уже рабочая, можно
   отправлять друзьям.

Готово: теперь при каждом `/app-push` Vercel автоматически пересобирает и обновляет сайт —
никаких дополнительных действий не требуется.

## (Необязательно) Приложения с базой данных

Если вашему приложению нужно хранить данные (записи, заявки, личные кабинеты), Claude
предложит подключить **Supabase** — бесплатный облачный сервис баз данных. Когда дойдёт до
этого шага:

1. Заведите бесплатный аккаунт на [supabase.com](https://supabase.com).
2. Создайте токен доступа на
   [supabase.com/dashboard/account/tokens](https://supabase.com/dashboard/account/tokens).
3. Вставьте токен в файл `.env.local` как `SUPABASE_ACCESS_TOKEN` (Claude подскажет, как).

Если аккаунта пока нет — ничего страшного: Claude всё равно соберёт рабочую версию
приложения, а базу можно подключить позже.

## Частые вопросы

**Мне нужно знать код?** — Нет. Вы только описываете, что хотите, и выбираете ответы.

**Где «живёт» приложение?** — Сначала на вашем компьютере (адрес `localhost:3000`). А чтобы
получить настоящий адрес в интернете — выполните Шаг 7 и подключите Vercel.

**Что-то пошло не так.** — Просто напишите Claude, что случилось, обычными словами — он
поможет. Можно повторить команду `/app-start` или `/app-build`.

**Можно поменять уже готовое приложение?** — Да. Просто напишите, что хотите изменить
(«сделай кнопку зелёной», «добавь страницу с ценами») — Claude внесёт правки. А чтобы быстро
поправить надпись — выделите её в окне справа и напишите `/app-change-text` с новым текстом.
